# Prompt Fragment Registry

This registry maps user request types to **source prompts**, **runtime verbatim prompt mirrors**, non-runtime notes/wrappers, owning skills, required inputs, output artifacts, export contracts, and review gates.

Runtime prompt execution must use `Runtime Verbatim Prompt`. Files under `skills/*/prompts/*.md` are non-runtime notes unless they are explicitly verified as byte/content-equivalent to the source prompt.

## Legacy-only prompt fragments

The following source prompts are reference material only and must not be selected by default runtime routes:

- `BIDV/Prompt/API/[Old]_API_Gen_TD_v1.2.txt`
- `BIDV/Prompt/API/[Old]_API_Gen_TC_From_TD.txt`
- API `.docx` variants that hardcode a target API instead of defining reusable BIDV workflow behavior.

Default API Test Design routing must use the `_BreakDown` prompt variants recorded in `knowledge/default/manifests/canonical-sources.yml` and `workflow-packs/default/canonical-sources.yml`.
