# BIDV Doc Normalizer

## Operating mode

Use this skill during BIDV knowledge bootstrap before runtime agents consume redacted context. Raw `BIDV/` files remain source evidence; runtime agents should use normalized and then redacted knowledge.

## Required inputs

- `knowledge/default/manifests/source-manifest.yml` generated by `scripts/build_source_manifest.py`.
- Source root containing the files listed in the manifest.
- Output folder, normally `knowledge/default/normalized/`.

## Workflow

1. Run `python scripts/normalize_docs.py --manifest knowledge/default/manifests/source-manifest.yml --output knowledge/default/normalized`.
2. Verify every manifest entry has a converted Markdown artifact or a conversion report.
3. For `.txt` and `.md`, preserve the source body after the metadata block.
4. For MHTML `.doc`, preserve headings and tables as Markdown.
5. For `.xlsx`, produce workbook profile Markdown and per-sheet TSV snapshots.
6. For `.pptx`, preserve slide numbers and extracted slide text.
7. For `.pdf`, produce a conversion report unless a PDF extractor is available.

## Output requirements

Each normalized Markdown file must begin with metadata:

- `source_path`
- `source_role`
- `canonical_status`
- `redaction_status: unredacted`

## Guardrails

- Do not use normalized output directly in runtime prompts until `secret-redactor` has produced redacted copies.
- Do not overwrite or modify source BIDV files.
- Do not publish raw sensitive files such as `Prompt/API/Gen Script/properties.txt`.
