# BIDV Legacy XLSX Exporter

## Operating mode

This skill is contract/tool-driven. It does not have a source prompt.

- Source BIDV Prompt: N/A
- Runtime Verbatim Prompt: N/A

Use the upstream testcase artifact metadata when the testcase source was generated by a BIDV prompt route.

## Selection criteria

Use this skill when the user asks to:

- export a BIDV legacy 19-column testcase workbook;
- create a VA-style manual execution workbook from Markdown or TSV;
- provide an XLSX artifact for tester manual update.

## Required inputs

- Legacy testcase Markdown table or legacy 19-column TSV.
- Output XLSX path.
- For tester review/handoff workbooks, the exporter must use formatted mode.

## Workflow

1. Verify the source artifact exists.
2. Normalize rows to `legacy_19_column_testcase_contract.md`.
3. Validate or preserve the legacy TSV source when available.
4. Export XLSX using `scripts/export_legacy_19col_xlsx.py --formatted` for all review/handoff workbooks.
5. Preserve manual execution columns for tester update.
6. Verify the workbook has wrapped cells and real Excel line breaks in step/data/expected-result cells.
7. Do not read or overwrite `BIDV/VA_19.004...xlsx`.

## Outputs

- Legacy 19-column testcase XLSX.
- Optional intermediate legacy TSV.

## Review gates

- Headers match the BIDV legacy 19-column contract.
- Manual execution/result columns are present.
- XLSX is derived from reviewed Markdown/TSV.
- Review/handoff XLSX was generated with `--formatted`.
- Step, data, and expected-result cells use real Excel line breaks, not literal `\\n` or `/n` text.
- Workbook has wrapping enabled for manual review readability.
- Missing business data remains `[PENDING_DOC]` or open questions.
- No external BIDV template workbook is modified.
